WordPress Security Checklist for Small Business Owners (2026)

March 28, 2026 · 8 min read

WordPress powers 43% of all websites on the internet. That makes it the single biggest target for hackers — and small business sites are the easiest prey.

Why? Because most small business WordPress sites are running outdated plugins, using weak passwords, and have zero security hardening. Attackers know this, and they use automated bots to scan thousands of WordPress sites per hour looking for exactly these vulnerabilities.

The numbers: Over 13,000 WordPress sites are hacked every day (Sucuri). 90% of all hacked CMS sites in 2025 were WordPress. The average cost of a small business data breach: $120,000 (IBM).

The good news? Most WordPress security issues are preventable with basic hardening. This checklist covers the 15 most important steps, ranked by impact.

🔴 Critical — Do These First

1. ✅ Update WordPress Core, Themes, and Plugins

Outdated software is the #1 attack vector. Go to Dashboard → Updates and install everything. Enable auto-updates for minor releases: add define('WP_AUTO_UPDATE_CORE', 'minor'); to wp-config.php.

2. ✅ Use Strong, Unique Passwords

Every admin account needs a password that's 16+ characters with mixed case, numbers, and symbols. Use a password manager (1Password, Bitwarden). Never reuse passwords across sites.

3. ✅ Install an SSL Certificate (HTTPS)

If your site still loads over HTTP, fix this immediately. Most hosts offer free SSL via Let's Encrypt. In WordPress Settings → General, update both URLs to https://. Then check for mixed content warnings.

4. ✅ Delete Unused Themes and Plugins

Every installed plugin is a potential attack surface — even if it's deactivated. If you're not using it, delete it. Keep your active plugin count as low as possible.

5. ✅ Set Up Automated Backups

If you get hacked, a clean backup is your lifeline. Use UpdraftPlus (free) to schedule daily database backups and weekly full backups to cloud storage (Google Drive, Dropbox). Test your restore process at least once.

🟡 Important — Do These Next

6. ✅ Enable Two-Factor Authentication (2FA)

Add 2FA to every admin and editor account. The free WP 2FA plugin works well. This stops 99% of brute-force login attacks dead — even if a password is compromised.

7. ✅ Limit Login Attempts

WordPress allows unlimited login attempts by default — an open invitation for brute-force attacks. Install Limit Login Attempts Reloaded (free) to lock out IPs after 3-5 failed attempts.

8. ✅ Change the Default Admin Username

If your admin account is still called "admin", create a new administrator account with a unique username, log in with it, and delete the "admin" account. Bots target "admin" by default.

9. ✅ Add Security Headers

Most WordPress sites have zero security headers. Install the Headers Security Advanced & HSTS WP plugin to add Strict-Transport-Security, X-Content-Type-Options, X-Frame-Options, and Content-Security-Policy with a few clicks.

10. ✅ Disable File Editing in the Dashboard

WordPress lets admins edit plugin and theme files directly from the dashboard. If an attacker gets admin access, this is the first thing they exploit. Disable it by adding to wp-config.php: define('DISALLOW_FILE_EDIT', true);

🟢 Recommended — Layer On These

11. ✅ Hide Your WordPress Version

WordPress broadcasts its version number in the HTML source by default. Attackers use this to target known vulnerabilities for specific versions. Remove it by adding to your theme's functions.php: remove_action('wp_head', 'wp_generator');

12. ✅ Move wp-config.php Up One Directory

WordPress automatically checks one directory above the web root for wp-config.php. Moving it there means it's not accessible via the web, protecting your database credentials.

13. ✅ Disable XML-RPC

XML-RPC is an old API that most sites don't need. It's a common target for DDoS and brute-force attacks. Disable it by adding to .htaccess: <Files xmlrpc.php> Order Deny,Allow Deny from all </Files>

14. ✅ Install a Security Plugin

Wordfence (free tier) provides a web application firewall, malware scanning, and login security in one package. It won't catch everything, but it's a solid baseline. Alternative: Sucuri Security.

15. ✅ Schedule Monthly Security Audits

Run a free scan of your site monthly to catch new issues — expired SSL, missing headers, new mixed content, or SEO problems that creep in with updates. Set a calendar reminder.

Bonus: Quick Security Wins

  • Use a reputable hosting provider. Shared hosting on a $3/month plan means you're sharing a server with hundreds of other sites — any one of which could be compromised. SiteGround, Cloudways, or WP Engine offer WordPress-specific security.
  • Use Cloudflare (free tier). Adds a CDN, DDoS protection, and the option to set security headers at the edge — all for free.
  • Monitor your site with Google Search Console. GSC alerts you if Google detects malware or security issues on your site.
