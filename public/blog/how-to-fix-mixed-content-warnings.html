How to Fix Mixed Content Warnings on Your Website (Step-by-Step)

March 28, 2026 · 6 min read

You installed an SSL certificate. Your site shows the padlock icon. But something's wrong — browsers are still flagging warnings, and your "secure" site doesn't feel so secure.

The culprit? Mixed content.

What is mixed content? It's when your HTTPS website loads some resources (images, scripts, stylesheets, fonts) over plain HTTP. The page itself is encrypted, but individual resources are not — creating a security gap that browsers warn visitors about.

Why Mixed Content Is a Problem

  • Browser warnings scare visitors. Chrome, Firefox, and Safari all show warnings or block mixed content entirely. Visitors see "Not fully secure" messages.
  • Google penalizes it. Mixed content is a negative ranking signal. Google wants fully encrypted pages.
  • It's a real security risk. An attacker on the same network can intercept or modify HTTP resources — injecting malicious scripts into an otherwise "secure" page.
  • Modern browsers block it. As of Chrome 86+, mixed content images are auto-upgraded to HTTPS, and mixed scripts/iframes are blocked outright. Your content may simply not load.

Two Types of Mixed Content

🔴 Active Mixed Content (Blocked by Default)

Scripts, stylesheets, iframes, and other resources that can change the page. Browsers block these automatically — meaning parts of your site may be broken without you knowing.

<script src="http://example.com/script.js"></script>
<link rel="stylesheet" href="http://example.com/style.css">
<iframe src="http://example.com/embed"></iframe>

🟡 Passive Mixed Content (Warning Only)

Images, audio, and video. Browsers show a warning but usually still load them (or auto-upgrade to HTTPS).

<img src="http://example.com/photo.jpg">
<audio src="http://example.com/track.mp3">

Step-by-Step Fix

Step 1: Find All Mixed Content

Open your website in Chrome and press F12 to open Developer Tools. Go to the Console tab. Mixed content warnings appear in yellow/red with the exact URLs that need fixing.

Look for messages like:

Mixed Content: The page at 'https://yoursite.com/' was loaded over HTTPS, but requested an insecure image 'http://yoursite.com/logo.png'.

Step 2: Update HTTP URLs to HTTPS

For each flagged resource, change http:// to https://. Common places to check:

  • Image URLs in your content (blog posts, pages)
  • CSS and JS files in your theme or template
  • Embedded videos or iframes
  • Font URLs (Google Fonts, custom fonts)
  • Third-party widgets (chat, analytics, maps)

Pro tip: Use protocol-relative URLs (//example.com/file.js) to automatically match the page's protocol — though explicit HTTPS is preferred.

Step 3: Fix Your Database (WordPress)

If you're on WordPress, old content often contains hardcoded http:// URLs from before you installed SSL. Use the free Better Search Replace plugin:

  1. Install and activate Better Search Replace
  2. Search for: http://yourdomain.com
  3. Replace with: https://yourdomain.com
  4. Select all database tables
  5. Run a dry run first, then execute

⚠️ Back up your database before doing this.

Step 4: Update Your .htaccess or Server Config

Force all HTTP traffic to HTTPS with a redirect:

Apache (.htaccess):

RewriteEngine On
RewriteCond %{HTTPS} off
RewriteRule ^(.*)$ https://%{HTTP_HOST}%{REQUEST_URI} [L,R=301]

Nginx:

server {
    listen 80;
    server_name yourdomain.com;
    return 301 https://$server_name$request_uri;
}

Step 5: Add the Safety Net Header

As a final catch-all, add this security header to automatically upgrade any remaining HTTP requests:

Content-Security-Policy: upgrade-insecure-requests

This tells browsers to silently upgrade any HTTP resource requests to HTTPS — catching anything you might have missed.

Quick win: If you use Cloudflare, enable "Always Use HTTPS" and "Automatic HTTPS Rewrites" in the SSL/TLS settings. This handles most mixed content issues automatically, no code changes needed.

How to Verify the Fix

  1. Clear your browser cache (Ctrl+Shift+Delete)
  2. Reload your site and check the Console in Developer Tools
  3. Look for the clean padlock icon — no warnings
  4. Run another free scan to confirm mixed content is resolved

Preventing Mixed Content in the Future

  • Always use HTTPS URLs when adding images, embeds, or third-party scripts
  • Update your WordPress Site URL to use https:// in Settings → General
  • Check third-party services — make sure any external widget, font, or API uses HTTPS
  • Set the CSP header with upgrade-insecure-requests as a permanent safety net
