E-commerce Website Security Checklist for 2026

By Steve · March 25, 2026 · 7 min read

If you sell anything online — whether on Shopify, WooCommerce, BigCommerce, or a custom platform — your website is a target. Not because you're special, but because you handle money and customer data. That's all attackers need to know.

The average cost of a data breach for a small business is $120,000. For an e-commerce store, that often means death. Customers don't come back after their credit card gets stolen from your site.

⚠️ Reality check: 43% of cyberattacks target small businesses, and e-commerce stores are hit disproportionately. You don't need to be Amazon to be a target — you just need to process payments.

The E-commerce Security Checklist

🔒 SSL & Encryption

  • HTTPS on every page — Not just checkout. Your entire site should be encrypted. Google penalizes HTTP sites in search results too.
  • HSTS header enabled — Forces all connections through HTTPS. Without it, attackers can intercept the initial HTTP request before redirect.
  • TLS 1.2 or higher — Older protocols (TLS 1.0, 1.1, SSL 3.0) have known vulnerabilities. Your host should support TLS 1.3.

🛡️ Security Headers

  • Content Security Policy (CSP) — Prevents malicious scripts from injecting into your pages and stealing customer data at checkout.
  • X-Frame-Options — Stops attackers from embedding your site in a fake page to trick customers (clickjacking).
  • X-Content-Type-Options — Prevents browsers from misinterpreting uploaded files as executable code.
  • Referrer-Policy — Controls what information gets shared when customers click links on your site. Protects order IDs and session tokens from leaking.

💳 Payment Security

  • PCI DSS compliance — If you handle card data directly, you must comply with PCI standards. Using Stripe, PayPal, or Shopify Payments handles most of this for you.
  • Never store card numbers — Use tokenized payment processors. If you're storing raw card numbers anywhere on your server, stop immediately.
  • 3D Secure enabled — Adds an extra verification step for card payments. Reduces fraud and shifts liability to the card issuer.

🔐 Account & Access

  • Admin panel on non-obvious URL — Don't use /admin or /wp-admin. Change it to something custom. Bots scan default admin paths constantly.
  • Two-factor authentication on admin accounts — If someone guesses your admin password, 2FA is the last line of defense.
  • Strong password policy for customer accounts — Require 8+ characters with mixed types. Offer "forgot password" instead of storing passwords in plain text.
  • Rate limiting on login pages — Prevent brute force attacks. Lock accounts after 5 failed attempts.

🔄 Maintenance & Updates

  • Platform and plugins updated — Outdated WooCommerce plugins are the #1 attack vector for WordPress stores. Update everything monthly at minimum.
  • Remove unused plugins and themes — Every unused plugin is an unlocked door. If you're not using it, delete it.
  • Regular backups — Automated daily backups stored off-site. Test your restore process quarterly.
  • Uptime monitoring — Know immediately if your store goes down. Downtime = lost revenue.

💡 Shopify users: Shopify handles SSL, PCI compliance, and server security for you. But you're still responsible for app permissions, admin access security, and any custom code you've added. Don't assume "Shopify handles it" covers everything.

The 5 Most Common E-commerce Security Mistakes

  1. Running outdated WordPress/WooCommerce — The #1 way stores get hacked. One unpatched plugin with a known vulnerability is all it takes.
  2. No security headers — Most stores have zero security headers configured. This makes cross-site scripting and clickjacking trivially easy.
  3. Weak admin credentials — "admin/password123" on a store that processes payments. We see it more often than you'd think.
  4. Third-party scripts without CSP — Every chat widget, analytics tool, and social plugin you add is a potential injection point. CSP limits what scripts can run.
  5. No monitoring — Most store owners don't know they've been breached until a customer complains. Set up alerts for unusual activity.

What Platform-Specific Security Looks Like

Shopify

  • SSL and PCI handled automatically ✅
  • Review app permissions — remove any app you're not actively using
  • Enable staff 2FA for all admin accounts
  • Use Shopify's built-in fraud analysis before fulfilling orders

WooCommerce

  • YOU are responsible for SSL, hosting security, and updates
  • Install a security plugin (Wordfence or Sucuri)
  • Move wp-admin to a custom URL
  • Keep WordPress core, WooCommerce, and ALL plugins updated
  • Use managed WordPress hosting (Kinsta, WP Engine) for better server security

Custom/Headless

  • Implement all security headers manually
  • Use a WAF (Web Application Firewall) like Cloudflare
  • Regular penetration testing
  • Ensure API endpoints are authenticated and rate-limited
